NEW TENANT PROVISIONING FLOW

1
Customer signs contract → triggers provisioning webhook
2
Pulumi spins up: ECS service + task, Neon DB branch, S3 bucket, Route 53 subdomain (~3 min)
3
Prisma runs migrations on fresh DB + seeds MAS policy templates
4
Clerk creates org, sends Admin invite to firm's IT contact
5
Customer accesses acme-bank.yourplatform.com — fully isolated environment, ready to use
Fully automated. Zero manual steps from contract to live environment.

CI/CD — SHIPPING UPDATES TO ALL TENANTS

GitHub Actions pipeline:
Push to main → TypeScript typecheck + lint
→ Unit + integration tests
→ Docker build → push to ECR
→ Pulumi updates all ECS task definitions
→ Rolling deploy — one tenant at a time, zero downtime
→ Prisma migrations run per tenant DB on startup
→ Sentry release tagged for error tracking
One deploy ships to all customers simultaneously — same Docker image, isolated DBs. DB migrations are backwards-compatible so rollbacks are safe.
