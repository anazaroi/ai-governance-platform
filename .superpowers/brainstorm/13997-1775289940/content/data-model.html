Core Data Model

Section 2 of 5 — The key entities and how they relate. Every module reads and writes to these.

CORE ENTITY

AIModel

id · name · description
type (LLM / ML / RPA / Rules)
status (Draft / Active / Retired)
businessUnit · owner
deployedAt · lastReviewedAt
currentRiskTier → RiskTier
vendorId → Vendor (nullable)
useCaseId → UseCase

UseCase

id · name · description
regulatoryCategory
(Credit / AML / Trading / etc.)

One AIModel → one UseCase

Vendor

id · name · type
(Internal / Third-party)
country · contractRef
dueDiligenceStatus

Nullable — internal models have no vendor

RiskAssessment

id · modelId · assessedAt
assessedBy · methodology
scores (JSONB — per dimension)
tier (High / Medium / Low)
rationale · nextReviewDate

One model → many assessments over time

WorkflowInstance

id · modelId · type
(Onboarding / Change / Retirement)
status (Pending / InReview / Approved / Rejected)
initiatedBy · currentStepId
dueDate · completedAt
↓ WorkflowStep
stepOrder · assignedTo · decision · decidedAt · comments

Policy

id · name · masReference
category · version
applicableTiers (array)
↓ Control
id · policyId · description · frequency · evidenceRequired
↓ ControlEvidence
modelId · controlId · fileUrl · status · reviewedAt

AuditEvent

Immutable append-only log — written on every state change across all modules

id · timestamp · actorId · actorRole · action · entityType · entityId · before(JSONB) · after(JSONB) · ipAddress
